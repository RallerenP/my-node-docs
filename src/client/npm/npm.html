NPM

What is NPM?


A major part of the Node ecosystem, NPM (Node Package Manager) was created as a way to share code among developers.

There's no need to re-invent the wheel every time you need to do something. Instead, use a library or dependency which can do the heavy lifting, leaving you to focus on the problem specific to your project.

NPM is here to help you accomplish this. It helps with managing dependencies, called 'packages', for your project.

How does NPM work?

To keep track of your dependencies, NPM makes use of a package.json file in your project.

Read more about package.json.

Simply, when you run npm install NPM will read the contents of the package.json, and install the packages from the "dependencies": ... and "devDependencies": ... fields into a node_modules folder in your project. These modules can themselves contain package.json files, which NPM will resolve and install if needed.



Installing with NPM



NPM can also be used to install new dependencies automatically 

    
        npm -i package-name1 package-name2 --save


If --save-dev is used instead of --save, the packages will be installed as dev-dependencies.

Runnings scripts with NPM

The NPM command can also run scripts from the scripts field in package.json

app.js 

    
        console.log("Hello World");


package.json 
    
        ...
        scripts: {
            "start": "node app.js"
        },
        ...
    
                


    
        npm run start
        # Out: Hello World
