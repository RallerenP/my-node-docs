Data Types


JavaScript has inferred types and type-coercion, meaning you rarely explicitly define a datatype. 

    
    const foo = 6; // bar is inferred as a number.
    const bar = "baz"; // baz is inferred as a string.


Type Coercion


JavaScript will attempt to coerce data types to fit a context. 

    
    const foo = "6"; // foo is inferred as a string.
    const bar = 3; // bar is inferred as a number.

    const baz = foo + bar
    // baz = foo + bar = "6" + 3 = "6" + "3" = "63"
    // baz = "63"
In this case, bar is coerced into the context of a string, and is treated as such. This can also go the other way, so a string can become a number through type-coercion.
This might occasionally cause trouble though. "11" + 1 = 111, while "11" - 1 = 10
This happens because the + operator can be applied to a string as string concatenation, while the - operator can only be applied to numbers.


All Datatypes

Primitives:

    
    const num_foo = 6; // number
    const num_bar = 6.2; // number

    const str = "3"; // string
    const str_foo = '3'; // string

    const bool = true; // string
    let something; // undefined

Complex:

    
    const foo = {key: "object_value", bar: 21}; // Object
    const bar = ["array_value", 2, false] // Object (Arrays are objects)
    const baz = null; // Object (null is an object)
    const qux = function() { }; // function
