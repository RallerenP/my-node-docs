package.json



The package.json file is a manifest for your project. It stores configuration and metadata, and is used by npm to keep track of dependencies.

NPM

NPM uses the package.json to keep track of dependencies, and to run scripts. For the simplest usage with NPM, all that is needed is a "dependencies" field.

package.json 

    {
       "dependencies": {
            ...
        }
    }


The dependencies field carries keys for names of dependencies, and the values will the versions required.

package.json 
    {
       "dependencies": {
            "express": "^4.17.1"
        }
    }


This will ensure that the project requires a version of express higher than 4.17.1. Running npm install, will install all dependencies listed in the package.json

Read more about package.json and NPM here



Setup



While there are no strict requirements for a package.json (perhaps except dependencies), you'll often find the following:

package.json 

    {
        "name": ...,
        "version": ...,
        "repository": ...,
        "author": ...,
        "license": ...,
        "scripts": { ... }
        "dependencies": { ... }
        "devDependencies": { ... }
    }


These fields are often present because they are generated when using npm init to create the package.json

The package.json for this project was looks like this: package.json 
    {
          "name": "my-node-docs",
          "version": "1.0.0",
          "repository": "https://github.com/RallerenP/my-node-docs.git",
          "author": "Rasmus Povlsen ",
          "license": "MIT",
          "type": "module",
          "scripts": {
            "start": "node src/app.js",
            "start:dev": "nodemon --watch src --watch public --ignore public/css/styles.css -e js,html,css --exec \"cross-env PORT=8080 yarn start\"",
            "build:css": "postcss public/css/tailwind.css -o public/css/styles.css"
          },
          "dependencies": {
            "cors": "^2.8.5",
            "express": "^4.17.1"
          },
          "devDependencies": {
            "@fullhuman/postcss-purgecss": "^4.0.2",
            "autoprefixer": "^10.2.5",
            "cross-env": "^7.0.3",
            "nodemon": "^2.0.7",
            "postcss-cli": "^8.3.1",
            "tailwindcss": "^2.0.3"
          }
    }
