Autoreload



Manually stopping and starting your app everytime the source changes can be quite a bother.

Luckily, there are tools that will watch your files for changes, and reload your app.

Nodemon

The most popular tool for accomplishing this is nodemon.

Nodemon is an NPM package, that is very simple to setup and use. Nodemon can be installed both globally and locally for the project. The simplest is to install globally

Terminal 

    
        npm install -g nodemon


From there it's easy to monitor a single file, and reload it upon changes

Terminal 
    
        nodemon [file]


This will watch the single file, and rerun it with node every time it changes.

Nodemon has many options though. For more advanced use it's best to install locally as a dev-dependency

Terminal 
    
        npm install nodemon --save-dev


and then use it as a script in package.json

package.json 
    {
       "scripts": {
            "dev": "nodemon [options] [file]"
        }
    }


For example, the dev script for this projects looks like this:

package.json 
    {
       "script": {
            nodemon --watch src --watch public --ignore public/css/styles.css -e js,html,css --exec \"cross-env PORT=8080 yarn start\""
        }
    }


Running npm run start:dev will make nodemon watch the src and public folder in my project, but only files ending with .js, .html, and .css. Ignoring an auto-generated file called styles.css.

Then it runs a tool called 'cross-env', which you can read about here.
